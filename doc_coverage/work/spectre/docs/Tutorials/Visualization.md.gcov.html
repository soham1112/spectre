       5             : # Running and Visualizing {#tutorial_visualization}
       6             : 
       7             : ### Building an Executable from an Existing Source File
       8             : 
       9             : SpECTRE source files for evolution executables are located in
      10             : `src/Evolution/Executables`. Executables can be compiled by running the command
      11             : `make EXECUTABLE_NAME` where `EXECUTABLE_NAME` is the name of the executable
      12             : as defined in the `CMakeLists.txt` file located in the same directory as the
      13             : source file. For example, to compile the executable that evolves a scalar wave
      14             : using a three-dimensional domain, one runs the command:
      15             : `make EvolveScalarWave3D`, which then results in an executable of the same name
      16             : in the `bin` directory of the user's build directory.
      17             : 
      18             : ### Running an Evolution Executable
      19             : 
      20             : Each SpECTRE executable reads a user-provided YAML file that specifies the
      21             : runtime settings of the evolution. This is where options such as the Domain,
      22             : AnalyticSolution, TimeStepper, etc are supplied. Example input files are kept
      23             : in `tests/InputFiles`, and are written to provide a functional base input file
      24             : which the user can then modify as desired. Copy the executable and YAML file
      25             : to a directory of your choice. The YAML file is then passed as an argument to
      26             : the executable using the flag `--input-file`. For example, for a scalar wave
      27             : evolution, run the command:
      28             : `./EvolveScalarWave3D --input-file Input3DPeriodic.yaml`.
      29             : By default, the example input files do not produce any output. This can be
      30             : changed by modifying the options passed to `EventsAndTriggers`:
      31             : 
      32             : ```
      33             : EventsAndTriggers:
      34             :   ? EveryNSlabs:
      35             :       N: 3
      36             :       Offset: 5
      37             :   : - Observe:
      38             :         VariablesToObserve: [MassDensity]
      39             : ```
      40             : 
      41             : This will trigger an observation event for the Variable MassDensity every three
      42             : slabs after the fifth one. A successful observation will result in the creation
      43             : of H5 files whose names can be specified in the YAML file under the options
      44             : `VolumeFileName` and `ReductionFileName`. One volume data file will be produced
      45             : from each Charm++ node that is used to run the executable. Each volume data
      46             : file will have its corresponding node number appended to its file name.
      47             : Visualization of the volume data will be described in the next section.
      48             : 
      49             : ### 3D %Data Volume %Data In ParaView
      50             : 
      51             : A SpECTRE executable with observers produces volume and/or reduced data h5
      52             : files. An XDMF file must be created from the volume data in order to do
      53             : visualization using ParaView. To this end we provide the python executable
      54             : `GenerateXdmf.py` in the `tools` directory. `GenerateXdmf.py` takes two
      55             : arguments which are passed to `--file-prefix` and `--output`. The argument
      56             : passed to `--file-prefix` is the name of the H5 volume data, leaving out the
      57             : node number and extension. The argument passed to `--output` is the desired
      58             : .xmf file name, also without filename extension. Use `--help` to see a further
      59             : description of possible arguments. Open the .xmf file in ParaView and select
      60             : the `Xdmf Reader`, *not* the version 3 readers. On the left hand side of the
      61             : main ParaView window is a section named `Properties`, here you must click the
      62             : highlighted `Apply` button. ParaView will now render your volume data. If you
      63             : only wish to visualize a few datasets out of a large set, we recommended
      64             : unchecking the boxes for the datasets you wish to ignore under `Point Arrays`
      65             : before clicking `Apply`.
      66             : 
      67             : ### Helpful ParaView Filters
      68             : 
      69             : Here we describe the usage of filters we've found to better visualize our data.
      70             : Feel free to contribute to this section!
      71             : 
      72             : #### Removing Mesh Imprinting
      73             : You may notice what appears to be mesh imprinting on the data. The imprinting
      74             : effect can be removed by applying the `Tetrahedralize` filter. To apply the
      75             : filter select the `Filters` menu item, then `Alphabetical` and finally
      76             : `Tetrahedralize`.
      77             : 
      78             : #### Creating Derived Volume Data
      79             : New volume data can be created from the existing volume data using the
      80             : `Calculator` filter. In the `Calculator`'s text box, input a numerical
      81             : expression in terms of existing datasets evaluating to the desired
      82             : quantity. For example, a vector-valued velocity dataset can be created
      83             : from three scalar velocity component datasets using the expression
      84             : `velocity_x * iHat + velocity_y * jHat + velocity_z * kHat` and hitting
      85             : the `Apply` button. By default, this will create a new dataset `Result`.
      86             : The name of the new dataset can be changed by changing the name provided
      87             : to `Result Array Name` above the `Calculator` text box.
      88             : 
      89             : #### Visualizing Vector Fields Derived From Scalar Fields
      90             : Use the `Calculator` filter described above in "Creating Derived Volume Data"
      91             : to create a new vector-valued dataset. Once this is created, use the `Glyph`
      92             : filter and set the `Active Attributes` to the vector you wish to visualize.
      93             : Make sure that `Scale Mode` is set to `vector`.
      94             : 
      95             : ### Reproducibility of Results
      96             : 
      97             : Being able to reproduce the results of simulations is important for scientific
      98             : integrity and accuracy. To this end, SpECTRE encodes the source tree,
      99             : environment variables, and a record of the third party library versions into the
     100             : executables. From most executables these can be retrieved using the flags
     101             : `--dump-source-as filename_without_extension`, `--dump-paths`,
     102             : `--dump-library-version`, and `--dump-environment`. The flag `--dump-only` will
     103             : cause the program to exit after the information has been dumped. You can use the
     104             : `--help` flag to get more details on what flags are available. The flag
     105             : `--dump-paths` will print to screen the environment variables `PATH`, `CPATH`,
     106             : `LD_LIBRARY_PATH`, `LIBRARY_PATH`, and `CMAKE_PREFIX_PATH` at link time. The
     107             : flag `--dump-library-versions` will print to screen the contents of the
     108             : `LibraryVersions.txt` file generated by CMake. The flag `--dump-environment`
     109             : will print to screen the output of `printenv` at link time. Finally, the flag
     110             : `--dump-source-as filename_without_extension` will dump the archive
     111             : `filename_without_extension.tar.gz`, which contains the entire source tree at
     112             : link time. The combined above information should be enough to reproduce
     113             : simulations results to roundoff error as long as the third party libraries
     114             : used to build the executable have not changed on the system the simulation was
     115             : run on.
     116             : 
     117             : Often only data from the simulation is retained, not the entire executable. In
     118             : order to make simulations reproducible from only the HDF5 output, any HDF5 files
     119             : written with SpECTRE's HDF5 wrappers contain the info about the run, output of
     120             : `printenv` at link time, the contents of `LibraryVersions.txt`, and an archive
     121             : of the source tree. These are inside the `header.hdr` group with names
     122             : `SimulationInfo`, `EnvironmentVariablesAtCompilation`, `LibraryVersions`, and
     123             : `src.tar.gz`. The source archive can be extracted by running the command
     124             : ```
     125             : h5dump -d /src.tar.gz -b LE -o src.tar.gz /path/to/hdf5/file.h5
     126             : ```
     127             : You will need to make sure the argument to `-b` is the same endianness as your
     128             : computer. In the above example little endian is used.
